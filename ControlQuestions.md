# Контрольные вопросы
## 1 практическая работа:
### В каком файле обрабатываются HTTP-запросы?
* В файле `views.py` обрабатываются HTTP-запросы.

### Какой файл отвечает за маршрутизацию?
* За маршрутизацию отвечает файл `urls.py`

### Какая команда используется для создания проекта на Django?
* Команда: `django-admin startproject имя_проекта`

### Какая команда используется для создания приложения в проекте на Django?
* Команда: `python manage.py startapp имя_приложения`


## 2 Практическая работа
### В каком файле подключаем приложение в Django?
* Приложение подключается в файле `settings.py` в разделе `INSTALLED_APPS`.

### Для чего нужна функция render?
* Функция `render` используется для рендеринга HTML-шаблонов с передачей данных контекста.

### В какой папке сохраняем HTML-шаблоны?
* Шаблоны сохраняются в папке `templates` внутри приложения или в основной директории проекта.


## 3 Практическая работа

* Там мы создаём структуру данных, как таблицы в базе данных.

### Какая команда создаёт скрипт для миграции?
* `python manage.py makemigrations`

### Где хранятся скрипты для миграции?
* В папке `migrations` внутри приложения.

### Какая команда выполняет миграции?
* ``python manage.py migrate``


## 4 Практическая работа
### Что такое связь один-к-одному (One-to-One) и как она реализуется в Django?
* Связь один-к-одному означает, что одной записи в одной таблице соответствует ровно одна запись в другой таблице. В Django реализуется с помощью `OneToOneField`.

### Что такое связь один-ко-многим (ForeignKey) и как она реализуется в Django?
* Связь один-ко-многим означает, что одной записи в одной таблице может соответствовать множество записей в другой таблице. В Django реализуется с помощью `ForeignKey`.

### Что такое связь многие-ко-многим (Many-to-Many) и как она реализуется в Django?
* Связь многие-ко-многим означает, что каждая запись одной таблицы может быть связана с множеством записей другой таблицы и наоборот. В Django реализуется с помощью `ManyToManyField`.


## 5 Практическая работа
### В каком файле добавляется модель в админ-панель?
* Модели добавляются в файл `admin.py` внутри соответствующего приложения.

### С помощью какого декоратора регистрируем модель в админ-панели?
* Модель регистрируется с помощью декоратора `@admin.register(ИмяМодели)`.

### Зачем нужна админ-панель?
* Админ-панель позволяет управлять данными в моделях через веб-интерфейс, добавлять, изменять и удалять записи, а также просматривать данные и управлять ими более эффективно.


## 6 Практическая работа
### Чем отличается метод get() от filter()?
* Метод `get()` возвращает один объект, соответствующий заданным критериям. Если объектов больше одного, или ни одного, будет выброшено исключение.
* Метод `filter()` возвращает QuerySet (список объектов), соответствующий критериям.

### Как создается запрос по связям?
* Для создания запросов по связям в Django ORM используется двойное подчеркивание `__`. Например, для доступа к полю name модели Author через модель Book, нужно использовать `author__name`.


## 7 Практическая работа
### Для чего нужен шаблонизатор?
* Шаблонизатор в Django используется для генерации HTML-страниц на основе данных, которые поступают от серверного приложения. Он позволяет смешивать статические части страницы (например, HTML-разметку) с динамическими данными, переданными через представления (views). С помощью шаблонизатора можно:
  * Генерировать HTML на основе данных (например, вывести список товаров).
  * Использовать конструкции, такие как циклы и условия, для динамического отображения контента.
  * Упрощать управление макетами страниц с использованием наследования шаблонов.
  * 
Пример: в шаблонизаторе можно использовать такие конструкции как `{{ product.name }}`, чтобы выводить имя товара.

### Как наследовать HTML-шаблон?
Django позволяет использовать механизм наследования шаблонов, чтобы избежать дублирования кода. Для этого создаётся базовый шаблон, который может содержать общие элементы страницы (например, заголовок, меню), а затем другие шаблоны могут его расширять.
1. Создание базового шаблона:
   * В базовом шаблоне определяются общие элементы, такие как шапка и подвал страницы, а внутри него создаётся блок, который будет заменяться содержимым дочерних шаблонов.
```html
<!-- base.html -->
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{% block title %}Магазин{% endblock %}</title>
    <link rel="stylesheet" href="{% static 'css/styles.css' %}">
</head>
<body>
    <header>
        <nav>
            <a href="{% url 'home' %}">Главная</a> |
            <a href="{% url 'about' %}">О нас</a> |
            <a href="{% url 'products' %}">Товары</a>
        </nav>
    </header>
    <div class="content">
        {% block content %}{% endblock %}
    </div>
    <footer>
        <p>Все права защищены. Магазин электроники, 2024</p>
    </footer>
</body>
</html>

```

2. Наследование от базового шаблона:
* В дочернем шаблоне можно расширить базовый и изменить только ту часть, которая требуется для конкретной страницы:
```html
<!-- products.html -->
{% extends 'base.html' %}

{% block title %}Товары{% endblock %}

{% block content %}
<h1>Наши товары</h1>
<ul>
    {% for product in products %}
        <li>{{ product.name }} — {{ product.price }} руб.</li>
    {% endfor %}
</ul>
{% endblock %}

```

3. Чем отличаются `{{}}` от `{%%}`?
* {{ ... }} используется для вывода переменных в шаблоне. Внутри фигурных скобок указывается переменная или объект, который будет выведен на страницу. Например:
```html
<p>Имя товара: {{ product.name }}</p>
```
Это выведет значение переменной product.name.

* `{% ... %}` используется для выражений шаблонизатора, которые не выводят данные напрямую, а управляют логикой шаблона (например, циклы, условия, наследование шаблонов). Внутри `{% ... %}` можно писать инструкции, например:
* Циклы
```html
{% for product in products %}
    <li>{{ product.name }} — {{ product.price }}</li>
{% endfor %}

```

* Условия
```html
{% if products %}
    <ul>
        {% for product in products %}
            <li>{{ product.name }} — {{ product.price }}</li>
        {% endfor %}
    </ul>
{% else %}
    <p>Нет доступных товаров</p>
{% endif %}

```

* `{{ ... }}` — для вывода данных.
* `{% ... %}` — для управления логикой и структурой шаблона.



## 8 Практическая работа
### Для чего нужен аргумент name в функции `path`?
Аргумент `**name**` в функции `path` используется для присвоения уникального имени маршруту (URL). Это имя позволяет ссылаться на маршрут в других частях приложения, таких как шаблоны, представления и редиректы. Вместо того чтобы писать прямой URL (например, `shop/products/`), можно использовать имя маршрута, что делает код более гибким и устойчивым к изменениям.

Пример:
* В файле `urls.py`:
    ``path('products/', views.products, name='products')``

* В шаблоне можно создать ссылку на этот маршрут с помощью тега `{% url %}`:
``<a href="{% url 'products' %}">Товары</a>``

Преимущества использования `name`:
* Гибкость: Если URL изменится (например, с `/products/` на `/items/`), достаточно изменить его в `urls.py`. Все ссылки, использующие имя маршрута, будут автоматически обновлены.
* Читаемость: Использование имен маршрутов улучшает читаемость кода, так как имя может быть более интуитивным, чем сам URL.


### 2. Для чего нужна библиотека Pillow?

Pillow — это библиотека для работы с изображениями в Python. 
Она используется в Django для поддержки работы с изображениями, особенно с полем ImageField в моделях. 
Когда вы используете ImageField для загрузки изображений, Pillow позволяет Django обрабатывать эти файлы.

Основные возможности Pillow:
* Загрузка изображений: Поддержка различных форматов изображений, таких как JPEG, PNG, GIF, TIFF и других.
* Изменение изображений: Вы можете изменять размеры изображений, обрезать их, изменять формат и делать другие преобразования.
* Обработка изображений: Pillow может считывать и изменять метаданные изображений, управлять цветами и накладывать фильтры.
* Поддержка работы с файлами изображений в Django: Django использует Pillow для обработки загруженных пользователями изображений.

**Пример установки Pillow**: Чтобы использовать ImageField в Django, нужно установить библиотеку Pillow:

``pip install pillow``

## 9 Практическая работа
### 1. Что такое слаг и зачем он нужен?
Слаг — это строка, состоящая из букв, цифр и дефисов, которая используется для формирования URL. 
Это делает URL более читаемыми для пользователей и улучшает SEO. 
В Django слаг обычно используется вместо идентификатора записи.

### 2. Как в Django передать динамические данные через URL?
В Django динамические данные передаются через URL с использованием специальных путевых параметров, 
таких как `<slug:slug>` или `<int:id>`. Эти параметры передаются в представление для обработки.

### 3. Как работает slugify?
Функция slugify преобразует строку в формат, пригодный для использования в URL. 
Она удаляет недопустимые символы и заменяет пробелы дефисами.


## 10 Практическая работа
### 1. Какие http запросы существуют?
HTTP-запросы (или методы) используются для отправки запросов от клиента (браузера) к серверу и обозначают, какое действие необходимо выполнить. Основные HTTP-запросы:

* GET: Запрос данных с сервера. Используется для получения ресурсов (например, веб-страницы или данных из API).
* POST: Отправка данных на сервер для создания нового ресурса. Обычно используется при отправке форм.
* PUT: Обновление существующего ресурса на сервере. Используется для полной замены данных ресурса.
* PATCH: Частичное обновление существующего ресурса на сервере. Обновляет только изменённые поля ресурса.
* DELETE: Удаление ресурса с сервера.
* HEAD: Запрашивает заголовки ресурса, но не сам ресурс. Используется для проверки состояния ресурса.
* OPTIONS: Запрашивает у сервера, какие методы и заголовки поддерживаются для данного ресурса.
* TRACE: Выполняет обратный запрос для проверки пути запроса. Редко используется.
* CONNECT: Устанавливает туннель к серверу, часто используется для прокси-серверов.

### 2. Какие запросы может отдавать html файл?
HTML-файл сам по себе не может "отдавать" HTTP-запросы, но может инициировать их через формы или ссылки. 
Основные запросы, которые могут быть инициированы через HTML:
* GET:
  * Когда пользователь переходит по ссылке или запрашивает страницу, браузер выполняет HTTP-запрос типа GET. 
  * Например, все гиперссылки `(<a href="url">)` или обычные URL-адреса отправляют запрос GET.
  * Также формы могут отправлять GET-запрос, если это указано в атрибуте формы:
```html
<form method="GET" action="/search/">
  <input type="text" name="query">
  <button type="submit">Поиск</button>
</form>

```
* POST:
  * Формы в HTML могут отправлять данные на сервер с использованием метода POST. 
  Этот метод обычно используется для отправки данных, например, при регистрации или добавлении новых товаров. Пример формы, отправляющей POST-запрос:
```html
<form method="POST" action="/add-product/">
  <input type="text" name="name">
  <input type="number" name="price">
  <button type="submit">Добавить товар</button>
</form>

```

* PUT, PATCH, DELETE:
  * Эти методы не могут быть отправлены напрямую через стандартные HTML-формы. 
  Однако их можно реализовать с помощью JavaScript или специальных библиотек, таких как `fetch` или `axios`.

### 3. Для чего нужен каждый запрос?
* GET:
  * Используется для получения данных с сервера. Это наиболее часто используемый метод для отображения страниц, выполнения поисковых запросов и получения информации. Безопасный метод, так как он не изменяет данные на сервере.
* POST:
  * Используется для отправки данных на сервер. Например, для создания нового ресурса (новая запись в базе данных, отправка формы). В отличие от GET, запросы POST могут содержать данные, такие как файлы или большие формы.
* PUT:
  * Используется для полного обновления существующего ресурса. В отличие от POST, который создаёт новый ресурс, PUT заменяет существующий ресурс целиком.
* PATCH:
  * Используется для частичного обновления ресурса. Обновляет только указанные поля, не заменяя весь ресурс, как это делает PUT.
* DELETE:
  * Используется для удаления ресурса с сервера. Например, удаление товара из базы данных.
* HEAD:
  * Используется для получения только заголовков ответа от сервера без тела. Полезен для проверки ресурса перед его загрузкой.
* OPTIONS:
  * Используется для проверки того, какие методы и заголовки поддерживаются сервером для конкретного ресурса. Часто используется для CORS (Cross-Origin Resource Sharing).
* TRACE:
  * Используется для диагностики маршрута запроса, но редко используется на практике.
* CONNECT:
  * Используется для установки сетевого туннеля, например, для HTTPS-соединений через прокси-сервер.


## 11 Практическая работа
### 1. Что такое CSRF-токен?
* `csrf_token` — это токен, который используется для защиты от атак CSRF (Cross-Site Request Forgery). 
Он добавляется в форму для проверки, что запрос пришел от авторизованного пользователя.
### 2. Какие преимущества даёт декоратор login_required?
* Декоратор login_required защищает страницы, требующие аутентификации, 
перенаправляя неавторизованных пользователей на страницу входа.
